Pattern C — Deterministic-First

The architectural pattern behind the agentic documentation pipeline
Default to deterministic. Use LLM only where judgment is genuinely required and cannot be reduced to a rule.
LLMs are powerful but expensive, non-deterministic, and opaque. In a production pipeline, those properties compound. Pattern C constrains LLM usage to the stages where it earns its cost — and makes everything else a deterministic function of the input.
The three-layer structure
Pre-script
deterministic
  • Input validation
  • Variable extraction
  • Template selection
  • Flag deduplication
  • Context assembly
  • Unit-testable
Bounded agent
LLM · constrained
  • Single bounded task
  • Receives clean context
  • No full pipeline state
  • Produces one output
  • Confidence-scored
  • Integration-testable
Post-script
deterministic
  • Output validation
  • Schema enforcement
  • Envelope construction
  • Anomaly flagging
  • Audit log entry
  • Unit-testable
Applied examples
Intake triage — eight detectors, zero LLM fully deterministic
raw change event → detect_change_type detect_scope detect_audience detect_urgency → detect_notify_trigger detect_content_trigger detect_review_trigger detect_escalation_trigger → routing manifest
Routing is the most consequential decision in the pipeline — a wrong route propagates through every downstream stage. Eight deterministic detectors make every routing decision auditable and unit-testable. uncertain is a first-class output: ambiguous signals escalate to confidence gating rather than forcing a wrong binary answer.
Communications drafting — pre-script + two sub-agents + post-script Pattern C applied
raw event payload → pre-script · template selection · variable extraction → sub-agent A · draft generation → sub-agent B · voice normalisation → post-script · envelope construction → approval gate
The pre-script extracts and structures exactly what the drafting agent needs — no noise, no irrelevant fields. Drafting and voice normalisation are separate agents because they are different cognitive tasks; a single agent asked to do both tends to over-normalise during drafting or under-normalise during review. The post-script constructs the output envelope deterministically — no LLM needed for that.
Quality review — three parallel sub-agents, one rule category each parallel · bounded
approved content → prose sub-agent · 39 rules style sub-agent · 19 rules terminology · 19 rules → post-script · consolidate findings → findings payload · rule IDs · severity
77 rules across three categories. A single agent evaluating all 77 rules faces context window pressure and rule interference. Three bounded sub-agents run in parallel — total latency equals one sub-agent call, not three sequential calls. Each finding is tagged with a rule ID, location, and confidence score. The post-script consolidates into a single structured payload.
The Pattern C test
Can you write a unit test for this decision?
YES → deterministic
The decision is a function of the input. Write it as a rule-based detector or script. It will be faster, cheaper, and fully auditable.
NO → consider LLM
The decision requires reading and understanding context. A bounded LLM agent is justified. Keep it narrow — one task, clean inputs, confidence-scored output.
