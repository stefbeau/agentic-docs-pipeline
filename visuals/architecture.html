Infrastructure Layer
MCP · Agent Platform · Vector Index · Dashboard
The foundation the pipeline runs on. MCP (Model Context Protocol) exposes live tools — page retrieval, corpus search, metadata access — to agents at runtime. The agent platform manages Skills-type agent deployment, A2A coordination, and prompt versioning. The vector index powers semantic search across the corpus. The dashboard provides the writer interface for human approval and pipeline monitoring.

Deterministic Layer
Pre-scripts · Detectors · Post-scripts · Zero LLM
All pipeline work that does not require reasoning lives here. Eight rule-based detectors handle intake triage. Pre-scripts extract and structure inputs before agents fire. Post-scripts validate outputs and construct handoff envelopes. Every function in this layer is unit-testable and produces the same output for the same input every time.

Agent Layer
Bounded LLM agents · Skills-type · One job each
Five LLM agents handle the work that genuinely requires judgment: semantic search and relevance scoring, content drafting, communications writing, voice normalisation, quality evaluation. Each agent has a single bounded responsibility and receives only the context it needs — not the full pipeline state. No agent communicates directly with another.

Human Approval Gate
Mandatory · Architectural · Not bypassable
The pipeline stops here. The writer reviews the impact manifest, content DIFF package, and communications draft. Each component requires an explicit approve / edit / reject decision before anything proceeds. The gate is not a UX feature — it is load-bearing architecture. It is what makes the pipeline trustworthy enough to operate at all.

Governance Layer
Audit log · Anomaly detection · Corpus change record
Every pipeline run produces a complete, structured audit log. Anomaly detection flags runs outside baseline thresholds. The corpus change record is append-only — every published update is traceable to the product change event that triggered it and the writer who approved it. Pipeline health metrics accumulate across runs to surface prompt tuning and maintenance signals.
run log · structured JSON anomaly detection · thresholds corpus change record · append-only health dashboard approval edit rate metrics
Pattern C — Deterministic-first
Default to determinism. Use LLM only where judgment cannot be reduced to a rule. The intake triage layer uses zero LLM — eight rule-based detectors make every routing decision auditable, unit-testable, and fast. LLM compute is reserved for bounded judgment tasks: drafting, scoring, evaluating.
Human-in-the-loop by design
The approval gate is not a safety net added after the fact — it was the first design constraint. Agents produce proposals; humans own decisions. This is what makes autonomous documentation pipelines trustworthy enough to run against a live production corpus.
Bounded agent scope
Each agent has one job and receives only the context it needs. No agent knows the full pipeline state. This makes individual agents independently testable and replaceable. When something breaks, you know which agent broke it — and you can fix it without touching the others.
Skills-type over Graph-type
Skills-type agents execute purely from the system prompt. Graph-type agents accumulate memory across turns and hallucinate in long pipelines. Re-architecting from Graph to Skills produced a 78% token reduction (165k → 35.5k per run) while preserving all pipeline behaviour.
